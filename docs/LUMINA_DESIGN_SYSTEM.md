# Lumina Design System

**Versão:** 1.0  
**Status:** Em evolução  
**Produto:** Lumina — Sistema Operacional da Escola

---

## 1. Propósito

O Lumina Design System define os padrões visuais, comportamentais e de experiência utilizados em toda a plataforma.

Seu objetivo é garantir que cada módulo da Lumina:

- seja simples de entender;
- mantenha consistência visual;
- transmita organização e confiança;
- seja acessível;
- permita evolução sem retrabalho;
- devolva tempo para quem administra e educa.

A interface deve parecer leve mesmo quando houver milhares de alunos, turmas, tarefas, documentos e registros.

---

## 2. Princípio central

> A Lumina devolve tempo para educar. A burocracia, a Lumina resolve.

Toda decisão de interface deve responder:

1. Isso facilita a compreensão?
2. Isso reduz trabalho?
3. Isso destaca o que merece atenção?
4. Isso mantém a identidade da Lumina?
5. Isso funciona em diferentes tamanhos de tela?

Quando a resposta for negativa, o elemento deve ser simplificado ou removido.

---

## 3. Personalidade visual

A Lumina deve transmitir:

- clareza;
- calma;
- inteligência;
- confiança;
- organização;
- modernidade;
- proximidade.

A Lumina não deve parecer:

- infantil;
- excessivamente colorida;
- carregada;
- tecnológica de forma exagerada;
- semelhante a um chatbot;
- um painel administrativo genérico.

---

## 4. Hierarquia das páginas

As páginas principais devem seguir, sempre que possível, esta estrutura:

1. Cabeçalho da página
2. Explicação curta
3. Ação principal
4. Indicadores
5. Filtros ou busca
6. Conteúdo principal
7. Ações rápidas
8. Orientação contextual da Lumi
